## How it ran

1. Dumped every broken recipe body to JSONL with `repair-dump-tmp.ts`,
   plus the 803-row master ingredient catalogue with
   `repair-dump-master-tmp.ts`.
2. Extended `repair-recipe-ingredients.ts` with `--count-blocks` and a
   `blocks[]` payload form so multi-block recipes (the ones with H3
   sub-sections like "For the dough" / "For the filling") could be
   repaired in place, keeping the existing block layout.
3. Smoke-tested 5 recipes by hand to verify the pipeline end-to-end
   (afghans-biscuits through angel-food-cake).
4. Split the remaining 386 recipes into 5 chunks and ran 5 Sonnet
   sub-agents in parallel, each producing a payloads JSONL of slug +
   amount + unit + prepNote + groupLabel decisions.
5. Consolidated each chunk's payloads file and applied via the
   throwaway `repair-apply-batch-tmp.ts` (one transaction per recipe).

## Patterns worth flagging back into the pipeline

- The bulk of body tokens were already canonical master slugs (great).
  The recurring mismatches were a small set of orientation differences
  — `ground-cinnamon` vs `cinnamon-ground`, `free-range-eggs` vs
  `eggs`, `fine-sea-salt` vs `sea-salt-fine`, `bramley-apples` vs
  `apple-bramley`. The slug-cheatsheet (kept at
  `C:/Users/Rebecca/AppData/Local/Temp/slug-cheatsheet.md` for the
  session) lists all of them. If the next pipeline build can
  normalise tokens at author time using the same map, future batches
  won't drift this way.
- Multi-block recipes (92 of 391) are common enough that the
  block-aware repair path is worth keeping in the script. The new
  `--count-blocks` and `blocks[]` payload form lives in
  `packages/db/scripts/repair-recipe-ingredients.ts`.
- A handful of cooking recipes (gumbo, croque-madame, navarin) needed
  brand-new master slugs (andouille-sausage, ham-cooked, stock-lamb)
  — added in this pass.
